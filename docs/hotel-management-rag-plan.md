# 酒店集团职能管理 RAG 工作台规划

## 规划定位

本规划以酒店集团的审计、人力、行政、财务总负责人视角设计。系统内置内容均为通用规划样本，正式使用时应由企业有效文件替换。

当前阶段不处理登录、角色和数据权限，先保证四个职能板块的分析、知识对照、历史追溯和管理闭环成立。

## 总负责人共同视角

总负责人收到的不是单份材料摘要，而是四层对照结果：

1. 当前事实：材料中的金额、主体、时间、流程、异常和缺失项。
2. 治理依据：公司章程、授权清单、部门制度、业务规范和会议决议。
3. 历史记忆：同类案件、既往发现、整改结论和重复发生的问题。
4. 管理动作：需补证、需复核、需整改、需升级决策的事项及证据定位。

知识效力默认按“公司章程及授权依据 > 已生效部门制度 > 业务规范/会议决议 > 历史案例”展示。检索相关度只用于召回，不代表文件效力，也不自动形成最终结论。

## 四个板块

### 审计与合同

- 集团授权边界：重大投资、担保、关联交易、预算外事项和合同审批层级。
- 酒店采购审计：工程、食品原料、客用品的准入、询比价、验收、付款及关联关系。
- 经营收入审计：PMS、POS、OTA、宴会合同、发票和资金流水的一致性。
- 高频异常：拆单、异常折扣、免单冲销、夜审差异、先付款后验收和历史整改复发。

### 人力管理

- 组织与编制：总部与单体酒店编制、人工成本率、关键岗位空缺和用工结构。
- 劳动用工：合同、岗位、工作地点、排班、考勤、加班补休和跨酒店调派的一致性。
- 关键岗位任用：财务、采购、工程、信息系统岗位的资格、背景核验和利益冲突声明。
- 人才经营：招聘质量、试用期评价、流失原因、劳动争议和同类历史问题。

### 行政管理

- 制度治理：版本、生效日期、适用范围、冲突条款、例外机制和定期复审。
- 印章与档案：用印审批、定稿一致性、空白用印、合同档案和证照原件。
- 酒店安全运营：消防、食品经营、特种设备、公共场所证照和外包施工。
- 决议执行：会议事项的责任部门、负责人、期限、验收标准和逾期升级。

### 财务管理

- 预算经营：按酒店、部门、科目和月份分析预算、滚动预测及历史同期。
- 酒店指标：入住率、平均房价、RevPAR、餐饮毛利、人工成本率和能耗偏差。
- 收入资金：客房、餐饮、宴会、会员及渠道结算的收入完整性和资金到账。
- 费用支付：预算、合同、发票、验收、审批阈值、职责分离和重复报销。

## 已落地工作流

```text
业务材料归档
  -> 确定性提取/计算与风险规则
  -> 按本板块 + 公司级范围检索知识资料
  -> 返回制度/章程/历史案例的版本、定位、原文和相关度
  -> 保存分析报告与引用证据
  -> 负责人复核并将案件标记为已确认/已关闭
  -> 将摘要、复核状态和发现项沉淀为历史案件记忆
  -> 下一次同类分析召回历史问题
```

知识资料库支持 PDF、DOCX、TXT、Markdown、CSV 和 Excel，也支持直接粘贴正文。资料按板块、类型、版本、生效日期和来源管理。历史案件按案件编号幂等更新，重新执行不会产生重复记忆。

## 分阶段建设

### 当前版本

- 七类资料分析流程接入统一知识对照。
- 公司章程、部门制度、业务规范、会议决议和历史案例统一检索。
- 本地同义词扩展、TF-IDF 和中文字符 n-gram 混合召回，无模型时可运行。
- 报告保存知识引用、效力层级、版本、原文定位、相关度和具体发现关联。
- 提供九份酒店集团规划基线样本和明确的非正式文件标识。

### 下一阶段

- 接入企业正式制度目录及文件状态同步，建立废止、替代和冲突关系。
- 增加审计项目底稿、酒店经营日报、薪酬编制和资金日报的专用工作流。
- 接入本地 BGE-M3 等嵌入模型及 reranker，形成向量、关键词、元数据三路召回。
- 将人工复核结论写回历史记忆，区分“模型发现、负责人确认、整改完成”。
- 建立跨酒店对标口径、异常趋势和重复问题专题看板。

### 后续治理

- 知识责任人、复审周期、版本冲突和引用有效性检查。
- 正式身份认证、板块与酒店数据权限、敏感字段隔离。
- 数据库备份、不可篡改审计日志和模型评测基线。

## 验收指标

- 每份分析可回到原始业务材料和知识条款原文。
- 同一份制度更新后不再召回旧版本内容。
- 同类案件能够召回已确认的历史问题，且不与制度依据混为同一效力等级。
- 财务权威数字继续由本地规则计算，RAG 和模型不得覆盖。
- 无外部模型、无网络时，资料解析、规则分析、知识检索和历史沉淀均可完成。
