# ADR-0003: 企业工作台采用统一持久化工作区与本地优先模型网关

## 状态

已采纳

## 背景

系统从合同审阅扩展到人力、行政和财务。业务人员通过浏览器操作，并要求每次上传和结果在下次打开时仍可查看。材料包含员工身份、工资、简历、发票和预算等敏感信息。

## 决策

1. 所有专业流程通过 `ReviewWorkspace` 执行，保持固定的“归档、解析、规则、可选模型、持久化、报告”顺序。
2. 原始文件保存在案件目录，结构化元数据、执行、发现项、任务和人工复核保存在 SQLite。
3. 权威金额、日期、重复、限额和预算差异由本地确定性代码计算，大模型不得覆盖这些结果。
4. 领域模块不得直接调用外部模型，统一通过 `ModelGateway` 路由。
5. L2/L3 数据优先使用 Ollama；本地失败时不得静默外发。只有单次明确授权后，脱敏内容才可调用外部模型。
6. 录用/淘汰、合同签署、制度发布、费用审批/付款、预算变更和风险关闭必须人工确认。
7. Streamlit 作为 MVP 操作台，领域逻辑不依赖 Streamlit，后续可迁移到 FastAPI 和独立前端。

## 影响

- 页面刷新、服务重启和重新执行不会丢失案件材料与结果。
- 七个流程共享一套证据、发现项、报告、审计日志和任务模型。
- 未安装或未启动 Ollama 时，规则检查和财务计算仍可完整工作。
- SQLite 适合单机和小规模局域网 MVP；多用户并发、细粒度 RBAC 和正式审批流需要后续迁移到服务端数据库。
