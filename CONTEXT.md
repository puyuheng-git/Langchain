# 酒店负责人个人经营治理

面向单个酒店负责人的个人工作产品，帮助其理解经营情况、发现管理风险、形成行动并持续跟进。

## 产品定位

**酒店负责人个人经营治理工作台**:
垂直于酒店领域、供酒店负责人个人使用的经营分析与管理辅助产品。
_Avoid_: AI 审计助手、企业智能管理工作台、酒店集团管理平台

**酒店负责人**:
对单体酒店经营结果和综合管理承担责任的总经理、店总或同等角色。
_Avoid_: 普通员工、集团职能负责人、平台管理员

**个人工作空间**:
由一名酒店负责人维护其酒店资料、经营分析、风险发现和管理行动的私有工作范围。
_Avoid_: 企业协同空间、多租户平台、集团共享工作区

**每日经营驾驶舱**:
酒店负责人每天进入产品后的主工作入口，集中呈现经营结果、关键异常、管理重点和待跟进行动。
_Avoid_: 静态报表页、通用数据大屏、单纯聊天首页

**经营闭环**:
从经营数据中识别异常，深入核验原因，形成管理行动并跟踪结果的连续过程。
_Avoid_: 一次性分析、只读报告、无责任人的建议

**经营日报导入**:
酒店负责人将 PMS、POS 或财务系统导出的 Excel/CSV 数据一次性导入个人工作空间，作为每日经营驾驶舱的数据入口。
_Avoid_: 手工逐项填报、首版厂商 API 直连、截图识别

**全服务商务酒店**:
同时经营客房、餐饮、宴会或会议等业务，并由酒店负责人综合管理收入、成本、服务与风险的中高端商务酒店。
_Avoid_: 仅客房经营模型、经济型有限服务酒店、度假综合体

**全店经营总览**:
每日经营驾驶舱覆盖客房、餐饮、宴会或会议、人工成本、能耗、费用与预算，并让酒店负责人从全店结果下钻到具体异常。
_Avoid_: 仅客房看板、单部门报表、彼此割裂的指标页面

**经营关注事项**:
每日最值得酒店负责人投入注意力的异常、机会或逾期行动，应说明业务影响、判断依据、可能原因和建议动作。
_Avoid_: 仅合规风险、无优先级告警、脱离数据的 AI 建议

**经营基线**:
用于判断经营结果是否值得关注的酒店自身参照，包括预算或目标、去年同期、近期同星期、滚动预测和负责人设置的阈值。
_Avoid_: 单一行业均值、模型主观判断、来源不明的固定标准

**个人行动卡**:
由经营关注事项形成的个人管理动作，记录截止时间、预期改善目标、关联指标和完成结果，并在后续经营数据中复核效果。
_Avoid_: 无来源待办、只有完成状态的任务、无法验证效果的建议

**本地个人数据空间**:
默认保存在酒店负责人个人电脑、无需上传云端且离线可用的经营数据范围，可由负责人执行加密备份、恢复或迁移。
_Avoid_: 默认云端账户、企业共享数据库、未经确认的自动同步

**营业日**:
由 PMS 夜审结算确定的酒店经营归属日期，是日报、指标和同期比较的统一时间单位；它可能不同于自然日和文件导入日。
_Avoid_: 上传日期、文件修改日期、未经夜审口径确认的自然日

**日报版本**:
同一营业日经营数据的历次导入记录；最新确认版本参与驾驶舱计算，旧版本保留用于追溯差异。
_Avoid_: 静默覆盖、重复累计、无法追溯的文件替换

**经营时间窗**:
驾驶舱以昨日实际结果、今日经营重点以及未来七天和三十天预判组成连续视图，支持复盘、执行和提前干预。
_Avoid_: 仅昨日静态日报、脱离历史的单点预测、没有行动连接的趋势图

**在手预订**:
截至当前时点已进入 PMS、尚待入住或消费的有效未来预订，是未来房量和收入预判的事实起点。
_Avoid_: 最终入住结果、纯模型预测、未确认的销售意向

**预订增量曲线**:
某个未来营业日在临近到店过程中新增、取消和变更预订的历史规律，用于把在手预订推演为可解释的经营预测。
_Avoid_: 无历史依据的增长率、AI 主观估计、只看当前预订量

**管理工具箱**:
由经营关注事项触发的专项分析能力，包括合同、用工、制度、会议、费用和预算分析；分析结论必须回到关注事项或个人行动卡。
_Avoid_: 与驾驶舱并列的孤立模块、没有后续动作的文件审阅、通用办公工具集合

**十分钟经营闭环**:
酒店负责人导入当日数据后，在十分钟内完成昨日复盘、重点确认、行动安排和后续跟进设置的核心使用结果。
_Avoid_: 以报告篇幅衡量价值、以模型调用次数衡量价值、长时间配置后才能使用

## 核心概念

| 术语 | 定义 |
|------|------|
| **审计 (Audit)** | 独立检查财务记录/内部控制，确保准确性、合规性的系统化过程 |
| **审计程序 (Audit Procedure)** | 审计师执行的具体检查步骤，如检查、观察、询问、函证、重新计算等 |
| **审计底稿 (Audit Working Paper)** | 审计过程中形成的记录和证据，是审计结论的基础 |
| **分析性复核 (Analytical Review)** | 通过对比财务数据趋势和比率来发现异常波动的审计方法 |
| **风险识别 (Risk Identification)** | 在合同/财务资料中标记可能存在问题或不合规的条款/数字 |
| **结构化提取 (Structured Extraction)** | 从非结构化文档（如合同 PDF）中自动提取关键字段，转成结构化表格 |
| **被审计单位 (Auditee)** | 接受审计的组织或个人 |
| **实质性程序 (Substantive Testing)** | 收集直接证据验证账户余额/交易真实性的程序 |
| **控制测试 (Test of Controls)** | 评估被审计单位内部控制制度是否有效运行的测试 |

## 合同审阅领域

| 术语 | 定义 |
|------|------|
| **合同 (Contract)** | 双方或多方之间设立权利义务关系的法律文件 |
| **条款 (Clause/Term)** | 合同中的具体规定，如金额、期限、违约责任等 |
| **标的 (Subject Matter)** | 合同指向的对象，如货物、服务、工程项目等 |
| **违约责任 (Liquidated Damages)** | 一方违约时应向另一方支付的赔偿金额或比例 |
| **管辖权 (Jurisdiction)** | 约定争议解决法院的条款 |
| **关键条款异常 (Key Term Anomaly)** | 合同中某条款明显偏离行业惯例或对某方明显不利 |
| **签约方 (Contracting Party)** | 签署合同的各方当事人 |
| **合同金额 (Contract Value)** | 合同约定的价款总额 |
| **期限 (Term)** | 合同的有效期或履行期限 |
